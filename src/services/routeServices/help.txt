====================================================
🧩 ORGANIZING YOUR API CALLS IN REACT (OR ANY FRONTEND PROJECT)
====================================================

Instead of dumping all your API calls everywhere in the code,
organize them in a clean, modular, and scalable way.

----------------------------------------------------
🧠 THE CONCEPT — “ONE SERVICE PER DOMAIN”
----------------------------------------------------

Think of your app’s features or entities. For example:

----------------------------------------------------
Feature        | Example API Endpoints
----------------------------------------------------
Users          | /api/users, /api/users/:id, /api/login
Routes         | /api/routes, /api/routes/:id
Products       | /api/products, /api/products/:id
Orders         | /api/orders, /api/orders/:id
----------------------------------------------------

Each of these feature areas is called a “domain.”

So, the idea is simple:

✅ Create one service file per domain  
✅ Put all related API calls inside that file  
✅ Import and use those services in your components

----------------------------------------------------
📁 EXAMPLE PROJECT STRUCTURE
----------------------------------------------------

src/
 ├── api/
 │    ├── usersService.js
 │    ├── productsService.js
 │    ├── ordersService.js
 │    └── routesService.js
 ├── components/
 ├── pages/
 └── App.jsx

----------------------------------------------------
💡 BENEFITS
----------------------------------------------------
- Keeps code organized and easy to maintain
- Makes API logic reusable
- Simplifies debugging and testing
- Easier to add new features or endpoints later
